TECHNICAL CASE STUDY

심층 트러블슈팅 케이스

단순 수정이 아닌, 근본 원인 분석(RCA)부터 시스템 구조 개선까지

CRITICAL

SocketException: Too Many Open Files — FD Leak RCA

이미지 처리 로직의 InputStream 미종료로 인한 File Descriptor 점진적 증가, 서비스 전면 장애 유발

탐지 (Detection)

서비스 접속 불가 알림 → SocketException: Too many open files 에러 로그 확인

분석 (Root Cause Analysis)

ls -l /proc/<PID>/fd | wc -l 명령으로 FD 수가 ulimit(65535)에 근접함을 확인. 이미지 처리 코드에서 InputStream을 close() 없이 사용하는 패턴 식별 — GC 의존 방식으로 인해 FD 반환 지연 발생

조치 (Remediation)

InputStream close 처리 추가 , 파일 처리 로직 전반을 리팩토링.

검증 (Verification)

배포 후 /proc/<PID>/fd 수치 안정화 확인, 이후 동일 장애 재발 0건

Before / After 비교

현상 분석

INSERT 패턴 분석

최적화 조치

① STRAIGHT_JOIN 제거 → 옵티마이저 자율 실행 계획 위임 ② 건별 INSERT → Bulk INSERT 변환 ③ 조인 컬럼 인덱스 추가 및 커버링 인덱스 검토
